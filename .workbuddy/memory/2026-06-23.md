## Prompt Evolution 优化效果差根因分析（完成）

对项目全量代码做逐行审阅，定位「6 个优化器 12 变体测试准确率(0.78-0.82)均低于 baseline(0.85-0.86)」的根因。

### 关键结论
- **评测口径已排除**：优化器训练评估（Evaluator+AccuracyMetric）与 baseline 测试评估的单条预测逻辑完全一致（均 replace({input}) + temperature=0 + 严格相等），benchmark_news.py 已修复 train/test 混比问题。结论可信。

### 根因（按优先级）
1. **P0-1 APE 不含 initial_prompt**（最致命）：`ape/optimizer.py` 的 all_candidates 从初始化为空(L86)到只 extend 新候选(L116)，从不加入 initial_prompt，L136 强制在新候选中选最优 → 机制性必然劣化。其余 5 个优化器都含 initial。
2. **P0-2 格式约束丢失**（全部 6 个）：meta-prompt 只约束保留 `{input}`，不约束保留「只输出类别名称」和末尾「类别：」引导。LLM 改写后输出格式漂移（"财经"→"财经类"），AccuracyMetric 严格相等判错。OPRO/PromptBreeder/EVOPrompt/SPO 还主动鼓励"改输出格式/加step-by-step"。
3. **P1-1 中英不一致**：meta-prompt 全英文，任务/数据全中文；OPRO 的 _DEFAULT_META_FEWSHOT 是数学辅导示例，严重误导。
4. **P1-2 小样本过拟合**：train 200/test 100，统计噪声驱动选择偏差，train 略高 test 暴露。
5. **P1-3 变异算子破坏格式**：Breeder/EVO/SPO 的变异策略含"Change the output format specification"。
6. **P2**：APE num_iterations 默认1；示例用 dataset[:3] 而前3条全"财经"；padding 加前缀污染；DSPy bootstrap 用 `tgt in pred` 过松收集错误格式 few-shot。

### 修复方案（按性价比）
1. APE 补 initial_prompt 基准候选（5行，仿 opro:143-149）
2. 所有 meta-prompt 加输出格式保留约束
3. meta-prompt 中文化 + 替换 OPRO 数学 few-shot
4. 增大 train 样本或加 hold-out 验证集
5. 弱化变异算子对格式的破坏
6. 批量清理 P2（num_iters、分层示例采样、padding、bootstrap 匹配）
